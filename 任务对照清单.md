## 2. 当前整体状态

当前项目已经完成 5 个方法的原型化实现，并打通了以下基础能力：

- 统一 CLI 入口
- 统一配置文件驱动
- 统一输入输出目录
- Python 总控 + R 后端混合架构
- 服务器环境可运行

但从严格意义上说，当前 5 个方法并不都已经达到“标准工具版”或“生产级正式实现”。

可将当前状态概括为：

- `burden`：已实现，含基础版、`SKAT` 标准版与 `regenie` 后端
- `skato`：已实现，基于 `SKAT` 包，并新增 `regenie` 后端
- `haplotype`：已实现原型版
- `quant_assoc`：已实现原型版
- `gwas_overlap`：已实现原型版

## 4. 已完成内容

### 4.1 架构层

已完成：

- Python 作为统一命令行入口
- R 作为统计后端
- 配置文件驱动运行
- 输出目录按方法拆分

已完成文件：

- [config_utils.py](D:\Genos\生信工具分析\genetic_support_tool\scripts\python\config_utils.py)

### 4.2 方法一：Burden

已完成：

- `engine=base`
  - 样本级 burden score 计算
  - 连续/二分类回归
- `engine=skat`
  - 调用 `SKAT` R 包
  - 使用标准 `Burden` 集合检验
- `engine=regenie`
  - 调用 `regenie --step 2`
  - 支持原生 `PGEN` / `BED` 输入
  - 支持 mask-based burden test
- 输出结果表、样本分数表、元信息文件

已完成文件：

- [run_burden.py](D:\Genos\生信工具分析\genetic_support_tool\scripts\python\run_burden.py)

  - 支持原生 `PGEN` / `BED` 输入
- 输出结果表和元信息文件

已完成文件：

建议优先级：

- 高
